1 # SpacemanDMM
2 
3 **SpacemanDMM** is a suite of tools for working with [DreamMaker] codebases
4 and map files. Most prominently it features a map renderer which emulates many
5 in-game graphical enhancements not usually visible in the map editor, including
6 overlays and smoothing. These features currently assume [/tg/station13], but
7 are highly likely to work on downstreams or closely related codebases.
8 
9 [DreamMaker]: https://secure.byond.com/
10 [/tg/station13]: https://github.com/tgstation/tgstation/

12 ## Installation
13 
14 Binary releases for Linux and Windows are available on the [releases] page, but
15 may not always be up to date.
16 
17 To build locally, begin by [installing Rust][rust] or updating your existing
18 installation. SpacemanDMM is tested against stable Rust.
19 
20 For one-time installation, run `cargo install --git https://github.com/SpaceManiac/SpacemanDMM cli`
21 to install the `dmm-tools` binary to `~/.cargo/bin`.
22 
23 For development or easy installation of updates, clone this repository and run
24 `cargo build -p cli --release` to create `target/release/dmm-tools`. See the
25 [source readme] for a list of the available packages, or omit `--release` for a
26 debug build.

32 ## Usage
33 
34 Executables are placed in `target/release` or `target/debug` depending on build
35 type. The CLI binary is known as `dmm-tools`. For best results, run from the
36 directory containing `tgstation.dme`.
37 
38 Basic usage: `dmm-tools minimap _maps/map_files/BoxStation/BoxStation.dmm`.
39 
40 By default, output is saved to `data/minimaps`, which can be changed with the
41 `-o` flag. The `-e` flag, specified before the subcommand, can be used to load
42 a different `.dme` file. More detailed usage information is available in the
43 `--help` output.
44 
45 Support is currently provided in /tg/station13's coderbus (ping `SpaceManiac`)
46 and on GitHub's issue tracker. Pull requests are welcome.

48 ## Language Server
49 
50 SpacemanDMM also provides a [language server] providing intelligence for the
51 DreamMaker language. A [Visual Studio Code plugin][vsc] is available, and will
52 install released language server binaries automatically. Details on available
53 features are listed in the language server's [package readme][ls-readme].
54 
55 Use `cargo build -p dm-langserver` to build a local copy of the language server
56 for debugging or development purposes. The plugin can be configured to run a
57 locally built language server rather than the binary releases.

63 ## Recommendations
64 
65 The minimap output is a very large PNG (e.g. 9.3 MB for Box). You are strongly
66 advised to run the resulting file through image optimization software such as
67 `pngcrush`. The `--pngcrush` option to the `minimap` subcommand can do this
68 automatically in many cases, but is off by default for speed reasons.
